Trabalho de Fixação da Aula 04

Elabore um texto que discuta quais são as vantagens da adoção do padrão Model-View-Controller(MVC) em aplicações web. Cite, pelo menos, cinco vantagens. Não esqueça de comentar todas as vantagens citadas.

Resposta

Redução de complexidade no código: Permite a escrita de código mais simples e mais fácil de interpretar para os desenvolvedores. Cada parte do código tem sua função independente e pode sermodificada sem afetar as outras partes.

Aplicações mais fáceis de manter: É possível atualizar, testar e depurar um serviço sem que os outros sejam afetados.

Melhor especialização: É possível designar membros da equipe para tarefas específicas com base em seus conhecimentos e usar diferentes linguagens de programação aproveitando o melhor de cada linguagem.

Multiplataforma: É possível desenvolver para diferentes plataformas (visão) e todas consumirem o mesmo serviço de backend (controle e modelo).

Atualizações: É possível publicar uma nova versão do backend (controle) sem que a versão antiga seja desligada. Mantendo as duas versões até que todos os dispositivos de apresentação (frontend) sejam atualizados e passem a consumir apenas a nova versão do backend.

Em uma aplicação que implementa o padrão Model-View-Controller(MVC), é possível ter múltiplas visões que acessam as mesmas regras de negócios, por meio do Controller? Justifique sua resposta.

Resposta

Sim: é perfeitamente possível que diferentes dispositivos (smartphones, PCs, laptops, tablets etc) tenham uma aplicação que consome o mesmo backend sem problemas. Os mesmos dados serão acessados em todos os dispositivos independente da plataforma ou da linguam=gem usada no desenvolvimento dessas aplicações.

O padrão Model-View-Controller(MVC) só pode ser usado em projetos de aplicações web? Justifique sua resposta.

Resposta

Sim: As aplicações web são provavelmente as maiores beneficiadas como modelo MVC que permite que cada parte rode em uma máquina diferente e comuniquem-se atravez da rede.
